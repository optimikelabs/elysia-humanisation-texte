# Gate de registre

## Principe
Humaniser ne veut pas dire familiariser. Le texte doit sonner humain dans son contexte.

## Registres

### Pro sobre
Pour : emails client, documents commerciaux, pages de service, synthèses.
Faire : phrases claires, verbes concrets, peu d'effets.
Éviter : `ça pique`, punchlines, humour forcé, lyrisme.

### Stratégique
Pour : notes de réflexion, positionnement, arbitrage.
Faire : direct, précis, vivant, avec un peu de tension.
Éviter : professoral, manifeste automatique, triades héroïques.

### Post social
Pour : posts courts, newsletters brèves, replies, angles publics.
Faire : rythme, compression, tension, une idée nette.
Éviter : thread cliché, anaphores publicitaires, morale finale.

### SEO / page web
Pour : pages de cocon, landing, contenu pédagogique.
Faire : concret, utile, orienté lecteur.
Éviter : sur-promesse, adjectifs creux, sections génériques.

### Technique
Pour : code, docs, specs, méthodes.
Faire : précision, mots métier, exemples exacts.
Éviter : simplifier au point de perdre le sens.

### Créatif / littéraire
Pour : textes narratifs, intros sensibles.
Faire : images rares, détails concrets, respiration.
Éviter : pseudo-poésie IA, noms abstraits émotionnels.

## Question de contrôle
Après réécriture : "Est-ce qu'un humain dans ce contexte écrirait vraiment comme ça ?"
